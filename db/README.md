# `db/` — Supabase database for vocab practice

The 🎴 单词练习 self-test and the index 错题复习 store progress in a hosted
**Supabase** (Postgres) database with **per-user accounts** (username + password).
Supabase Auth is email-based, so the login bar synthesizes an address from the
username behind the scenes: `用户名` → `用户名@example.com`. Users never see or
type an email; the `@example.com` suffix is stripped again when their name is shown.
The static pages talk straight to Supabase from the browser using the public
**anon key**; privacy is enforced by **Row-Level Security** (each user reads/writes
only their own stats).

## Files

- **`schema.sql`** — tables, RLS policies, and functions. Run once.
  - `words` — catalog of every practiceable word (`slug, word, reading, meaning, type`).
  - `user_word_stats` — per-user aggregate (`seen, correct, wrong, score, last_seen`);
    **accuracy = `correct / seen`**. RLS: a user can only touch `auth.uid() = user_id`.
  - `record_attempt(slug, word, reading, meaning, type, correct)` — atomic upsert of one answer.
  - `import_stats(jsonb)` — one-shot import of legacy browser `localStorage` progress
    (the **导入本机旧记录** button).
- **`words_seed.sql`** — generated by `python3 work/export_words.py`; bulk-inserts the
  word catalog. Re-generate + re-import whenever lesson content changes.

## One-time setup

1. Create a project at [supabase.com](https://supabase.com).
2. **Auth** → Providers → enable **Email**, and turn **off** "Confirm email".
   This is **required**: the synthetic `用户名@example.com` addresses can't receive
   mail, so a confirmation step would lock every account out. With it off, sign-up
   logs the user straight in. (Tradeoff: no email means no self-service password
   reset — reset a forgotten password from the Supabase dashboard.)
3. **SQL editor** → run `db/schema.sql`.
4. `python3 work/export_words.py`, then run the generated `db/words_seed.sql` in the SQL editor.
5. **Project Settings → API**: copy the **Project URL** and **anon public** key into
   `supabase-config.js` (copy `supabase-config.example.js` first — it's git-ignored).
6. Serve the site over **http(s)**, not `file://` (auth/CORS need a real origin):

   ```sh
   python3 -m http.server 8000        # then open http://localhost:8000/
   ```

   …or deploy the static files to any static host (GitHub Pages, Netlify, …).

## Notes

- The **anon key is meant to be public**; never put the `service_role` key in the
  frontend. RLS is the security boundary — don't disable it.
- DB-only by design: practicing requires being logged in. The first time you log in
  on a device that has old `localStorage` progress, click **导入本机旧记录** to migrate it.
