抽出方法（再現手順）
----------------------------------------------------------------
1. pdftotext -enc UTF-8 -raw で内蔵OCR層を抽出（-raw は読み順を保つ）。
2. このPDFは各見開きを2回ずつ重複保存しているため、重複を除去。
3. 目次のページ番号と各章見出しの位置から、章の境界をPDFページ単位で確定。
   章が見開きの左頁から始まる箇所では、見開き内で見出し行を検出して分割し、
   前章末尾と新章冒頭を正しいファイルへ振り分けた。
4. 整形：文字間の不要な空白を除去し、折り返された行を段落へ連結。各頁の
   柱（ランニングヘッダ）とノンブルを除去。
   ※生成スクリプトは ../work/build.py （入力は pdftotext -raw 出力）。

----------------------------------------------------------------
既知の制約（重要）
----------------------------------------------------------------
・本抽出は元PDFのOCR層をそのまま整形したものです。本文（和文）の精度は
  概ね良好ですが、OCRの文字誤りは訂正していません。特に小さく密な箇所――
  漢文（白文）の引用、説話の漢字題名、ノンブル、注番号――には誤字が残ります
  （例：「三百十六」→「三百十二」、尹→ヂ、会→余 など）。
・縦書き右→左の読み順は -raw により概ね正しく復元されていますが、目次・索引・
  事件題目の一覧など、複雑なレイアウトの箇所では語の前後関係が乱れることが
  あります。22_索引・奥付.txt は特に断片化が大きく、見出し語とページ番号の
  対応は崩れています。
・見開きをまたぐ章境界では、同じ〔PDF p.N〕が隣り合う2ファイルに現れます
  （その見開きの右頁＝前章末、左頁＝新章冒頭を、それぞれのファイルへ分けたため）。
・表紙（PDF p.1）は画像のみでテキスト層がないため、00_前付.txt 冒頭の書誌は
  標題紙の記載から補ったものです。
・抽出した非空白文字は元（重複除去後）の約99%。差分は除去した柱・ノンブル分です。
